I. OVERVIEW
I.1. DEFINITIONS
Terms Description
Pay2Pay Pay2Pay Payment Gateway
Merchant Partners using the services of Pay2Pay
Bank The bank where the partner's customers want to receive money
HTTPS Secure Hypertext Transfer Protocol
TLS Transfer Layer Security
Signature To ensure data transmission between Merchant and Pay2Pay is secure and accurate, all
data transmitted between the two parties will be encrypted and digitally signed to ensure
data constraints.
I.2. DOCUMENT PURPOSE
The document describes the detailed steps necessary for partners to integrate into the Pay2Pay
Payment Gateway system.
I.3. OBJECT
This document is used for partner developers to integrate into the Pay2Pay Payment Gateway system.
II. COMMUNICATION MODEL
The following figure describes an overview of the communication model between the Merchant's
system and the Pay2Pay system.
- Pay2Pay uses HTTPS/TLS1.2 connection standard
- Every request sent from merchant must be authenticated by the JSON Web Token solution.
- Pay2Pay provides APIs according to Restful Webservice standards, transmitting and receiving data
in JSON format.
- To ensure data transmission and reception between the two parties is secure and authentic, all data
will be encrypted and digitally signed.

III.2. DIGITAL SIGNATURE
.
All data transmitted and received between the two parties will be encrypted and digitally signed
ALGORITHM:
- Signature is created based on RSA-2048 algorithm.
- Merchant creates RSA-2048 key pair - including private key and public key - through online tools
and sends Public key to Pay2Pay.
- Private key for merchant to create signatures.
- Public key provided by the merchant to Pay2Pay to verify the merchant's signature.
HOW TO CREATE A SIGNATURE:
Step 1: Filter and sort header parameters alphabetically (authorization, verification & fields prefixed
with p-) and concatenate header values.
Step 2: Take the header values and combine them with the request body to get the result of the string
to be signed
Step 3: Signature = SHA256withRSA(String to be signed in Step 2)

IV. SYSTEM INFORMATION

Environment URL
Test https://uat-api.pay2pay.vn
Production https://api.pay2pay.vn
IV.2. MERCHANT ADMIN WEB INFORMATION
Merchant will be provided with an account to login the Merchant Administration Page built by
Pay2Pay to track payment transactions.
Environment URL
Test https://uat-merchant.pay2pay.vn
Production https://merchant.pay2pay.vn
Merchant Administration Page login account information will be provided when the two parties
proceed with the integration.
IV.3. INFORMATION MERCHANT NEEDS TO PROVIDE
Merchant needs to provide the following information to Pay2Pay
Info Condtion Description
IP address Mandatory List of IP addresses wil be used to send requests to Pay2Pay.
- Required on Production environment
- Optional on Test environment.
ipn_url Mandatory URL to receive transaction results from Pay2Pay, sent
immediately after confirmation from the bank.
redirect_url Optional URL for Pay2Pay to transfer customers to the merchant site,
after the customer completes the transfer on the Pay2Pay
site.
Public Key Mandatory The public key the merchant provides to Pay2Pay allows
Pay2Pay to verify the merchant's signature when sending a
request to Pay2Pay.

public static string ComputeSha256HashBase64(string inputString)
{
byte[] inputBytes = Encoding.UTF8.GetBytes(inputString);
// Create a SHA256 instance
using (SHA256 sha256 = SHA256.Create())
{
byte[] hashBytes = sha256.ComputeHash(inputBytes);
string base64Hash = Convert.ToBase64String(hashBytes);
return base64Hash;
}
}

public static String computeSha256HashBase64(String inputString) {
try {
MessageDigest digest = MessageDigest.getInstance("SHA-256");
byte[] hashBytes = digest.digest(inputString.getBytes(StandardCharsets.UTF_8));
String base64Hash = Base64.getEncoder().encodeToString(hashBytes);
return base64Hash;
} catch (NoSuchAlgorithmException e) {
throw new RuntimeException(e);
}
}
