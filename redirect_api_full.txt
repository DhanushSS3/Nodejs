  SIGNATURE 
To ensure data transmission between Merchant and Pay2Pay is secure and authentic, all data transmission between the two 
parties will be encrypted and digitally signed. 
ALGORITHM: 
The signature is created based on the SHA256 hash algorithm 
HOW TO CREATE A SIGNATURE: 
The steps to create a signature are as follows: 
1.    Sort the data parameters in URL Query string order by alphabet.  
2.    Get the value of each sorted data field name and join them together.  
3.    Connect the above string with  merchant_key (merchant_key is the merchant key code)  
4.    Apply the SHA-256 hash algorithm to the concatenated string.  
Convert the hash result to Base64 format.

V.     LIST OF API STATUS CODES 
Status Codes indicate whether a  transaction has succeeded, failed, or is still being processed. 
Merchants use these codes to decide how to handle each transaction, such as confirming success, 
waiting for completion, or retrying when necessary.
 
No                Status                                                                             Description 
1              SUCCESS 
The transaction has been completed successfully. The final result is confirmed, and no 
further action is required. Applicable to both collection and payout.   
2          PROCESSING 
The transaction has been received and is currently being processed by the bank or 
payment provider. The final status will be updated later through callback.   
3                  FAIL 
The transaction could not be completed. This may be due to incorrect information, 
insufficient balance, exceeded limits, timeout, or external bank rejection. Final state   — no 
further processing will occur.  
4                  INIT 
The transaction has been initialized and recorded in the system, but no processing has 
started yet. This status indicates the initial creation stage before being sent for processing.

6              SUSPECT

or risk control rules. Further verification or manual review is required before determining 
the final result.

VIII.     CODE SAMPLE 
Sample code creates a signature based on the SHA256 hash algorithm 
1.    PHP  
function computeSha256HashBase64($inputString) {  
    // Compute the SHA-256 hash of the input string  
    $hash = hash('sha256', $inputString, true);      
    // Encode the resulting hash in Base64  
    $base64Hash = base64_encode($hash);      
    return $base64Hash;  
}  
  
2.    C#  
public static string ComputeSha256HashBase64(string inputString)  
    {          
        byte[] inputBytes = Encoding.UTF8.GetBytes(inputString);  
        // Create a SHA256 instance  
        using (SHA256 sha256 = SHA256.Create())  
        {  
            byte[] hashBytes = sha256.ComputeHash(inputBytes);  
            string base64Hash = Convert.ToBase64String(hashBytes);  
            return base64Hash;  
        }  
    }  
   
3.    ReactJs  
import CryptoJS from 'crypto-js';   
function computeSha256HashBase64(inputString) {  
    // Compute SHA-256 hash  
    const hash = CryptoJS.SHA256(inputString);  
   
    // Convert the hash to a Base64 string  
    const base64Hash = CryptoJS.enc.Base64.stringify(hash);  
   
    return base64Hash;  
}  
   
4.    Java  
public static String computeSha256HashBase64(String inputString) {  
        try {  
            MessageDigest digest = MessageDigest.getInstance("SHA-256");  
            byte[] hashBytes = digest.digest(inputString.getBytes(StandardCharsets.UTF_8));  
            String base64Hash = Base64.getEncoder().encodeToString(hashBytes);   
            return base64Hash;  
        } catch (NoSuchAlgorithmException e) {  
            throw new RuntimeException(e);  
        }  
    }
